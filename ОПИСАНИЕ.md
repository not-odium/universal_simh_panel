# SIMH Universal Panel — описание работы

Графическая передняя панель (front panel) для эмуляторов SIMH. Программа
показывает аутентичный пульт исторической ЭВМ с лампами, тумблерами, кнопками и
табло, а нажатия на панели превращает в текстовые команды SIMH и отправляет в
запущенный эмулятор. Ответы SIMH разбираются и отражаются на панели.

Внешний вид каждой из 10 машин задаётся **декларативно** в JSON-конфигах, а сам
код универсален — одна и та же логика обслуживает PDP-11, PDP-8, VAX и др.

---

## 1. Общая схема

```
   пользователь
        │  клик по кнопке/тумблеру
        ▼
  UniversalPanel ──сигнал──▶ MainWindow ──команда SIMH──▶ SIMHController ──stdin──▶ процесс SIMH
        ▲                        ▲                                                     │
        │ обновление ламп        │ разбор ответа (телеметрия)      stdout (фоновый поток)
        └────────────────────────┴─────────────────────────────────────────────────┘
```

- **UniversalPanel** строит панель из конфига и шлёт сигналы при действиях
  пользователя.
- **MainWindow** — «мозг»: запускает/останавливает эмулятор, переводит нажатия в
  команды SIMH, разбирает вывод и зажигает лампы/табло.
- **SIMHController** — управляет процессом эмулятора: запуск, отправка команд в
  stdin, чтение stdout в фоновом потоке.

---

## 2. Запуск эмулятора

- **Выпадающий список** сверху — выбор машины. Панель перестраивается под неё,
  рядом показывается описание.
- **«Запустить» (Ctrl+F5)** — стартует бинарник SIMH как дочерний процесс.
- **«Остановить» (Ctrl+F6)** — корректно завершает эмулятор.
- **«Пилюля» состояния** справа: зелёная «● работает» / серая «○ остановлен».

> На каждой ОС нужен свой бинарник SIMH в папке `simh/`: Windows — `.exe`,
> Linux/macOS — нативные сборки без расширения. GUI кроссплатформенный, а вот
> бинарники — под конкретную систему.

## 3. Элементы управления панели

### Регистр тумблеров (SR)
Ряд тумблеров, каждый = один бит. Кликом переключаете бит и набираете число
(читается восьмерично). Это число используют кнопки LOAD ADDR и DEP.

### Кнопки
| Кнопка | Что делает | Команда SIMH |
|--------|-----------|--------------|
| **LOAD ADDR** | заносит значение тумблеров в регистр адреса (локально) | — |
| **DEP** (deposit) | пишет значение тумблеров в память по текущему адресу, затем адрес +1 | `deposit <адрес> <значение>` |
| **EXAM** (examine) | читает память по текущему адресу, показывает данные, адрес +1 | `examine <адрес>` |
| **START** | запуск программы | `run` |
| **CONT** | продолжить | `continue` |
| **HALT** | остановить процессор | `halt` |
| **S.INST** | один шаг | `step 1` |
| **BOOT** | загрузка | `boot rk0` |
| **RESET** | сброс | `reset all` |

Адреса и значения передаются **восьмерично** (как на настоящем пульте DEC).
LOAD ADDR и DEP отражаются на панели сразу; EXAM и остальные — после ответа SIMH.

## 4. Индикаторы (лампы и табло)

- **RUN** — горит, пока процессор исполняет программу; гаснет при остановке
  (команда HALT, брейкпоинт или возврат к приглашению `sim>`).
- **POWER / MASTER** — горят, пока эмулятор запущен.
- **CONSOLE / стоповые лампы** — горят при остановке на консоли.
- **ADDRESS / DATA** — значение регистров на рядах ламп. Биты сгруппированы
  восьмеричными **триадами** (неполная группа — слева, как на реальном пульте),
  соседние триады окрашены в два чередующихся цвета.
- **Семисегментное табло** дублирует значение в **восьмеричном** виде
  (у VAX — шестнадцатеричное, у IBM 1401 — десятичное, т.к. это их родные
  системы счисления).

## 5. Терминал

Внизу — консоль SIMH. Можно вводить **любые команды SIMH напрямую**, например:

```
d 1000 5000     ; записать 5000 в адрес 1000
e 1000          ; посмотреть содержимое 1000
d pc 1000       ; установить счётчик команд
step            ; один шаг
e r0            ; посмотреть регистр R0
show cpu        ; сведения о процессоре
```

Вывод идёт в терминал и попутно разбирается панелью (обновляет лампы/адрес).

---

## 6. Поле «Регистр:» — чтение и запись

Работает для **всех** эмуляторов (имена регистров обрабатывает сам SIMH):

- **«Смотреть»** — вводите имя регистра (`PC`, `R0`, `SP`, `PSW`, …), шлётся
  `examine <имя>`, значение показывается в терминале.
- **«Записать»** — в поле после знака `=` вводите значение и жмёте «Записать»:
  шлётся `deposit <имя> <значение>`, затем регистр сразу читается обратно для
  подтверждения. Так значение попадает именно в **регистр процессора**
  (в отличие от кнопки DEP на панели, которая пишет в память по адресу).

---

## 7. Тема оформления

Меню **«Вид» → «Светлая/Тёмная тема»** переключает оформление на лету
(меню, панель инструментов, фон, терминал). Аппаратная панель остаётся тёмной в
обеих темах — как реальный металлический пульт.

---

## 8. Структура проекта

| Файл / папка | Назначение |
|--------------|-----------|
| `main.py` | точка входа: создаёт окно, применяет тему |
| `ui/main_window.py` | главное окно: жизненный цикл эмулятора, команды, телеметрия, тема, просмотр регистра |
| `ui/panel_widget.py` | `UniversalPanel` — сборка панели из конфига |
| `ui/indicators.py` | виджеты пульта: лампа, тумблер, табло, кнопка, ряды ламп/тумблеров |
| `ui/terminal_widget.py` | консоль SIMH |
| `ui/themes.py` | стили светлой/тёмной темы |
| `ui/theme_state.py` | палитра панели |
| `core/simh_controller.py` | управление процессом SIMH (запуск, ввод/вывод) |
| `core/config_loader.py` | загрузка конфигов |
| `config/simh_config.json` | список машин и путей к бинарникам |
| `config/panel_config.json` | раскладка панелей всех машин |

---

## 9. Запуск программы

```
pip install -r requirements.txt
python main.py
```

Для запуска самих эмуляторов положите в папку `simh/` бинарники SIMH под вашу ОС.
